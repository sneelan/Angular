List of Syntax

  1. ngIf: *ngIf="condition" - Conditionally renders content based on the truthiness of an expression.
  2. ngFor: *ngFor="let item of items" - Iterates over a collection and renders content for each item.
  3. ngSwitch: [ngSwitch]="value" - Conditionally renders content based on a matching value using multiple cases.
  4. ngClass: [ngClass]="{ 'class-name': expression }" - Binds CSS classes to an element based on the evaluation of an expression.
  5. ngStyle: [ngStyle]="{ 'property': expression }" - Binds inline CSS styles to an element based on the evaluation of an expression.
  6. ngModel: [(ngModel)]="value" - Binds the value of an input element to a property using two-way data binding.
  7. ngSubmit: (ngSubmit)="onSubmit()" - Binds a method to the submit event of a form element.
  8. ngClassEven/ngClassOdd: [ngClassEven]="expression" [ngClassOdd]="expression" - Binds CSS classes dynamically to elements in an ngFor loop based on even or odd index.
  9. ngSwitchCase: *ngSwitchCase="value" - Defines a case for the ngSwitch directive.
  10. ngSwitchDefault: *ngSwitchDefault - Defines a default case for the ngSwitch directive.
  11. ngStyleEven/ngStyleOdd: [ngStyleEven]="expression" [ngStyleOdd]="expression" - Binds inline CSS styles dynamically to elements in an ngFor loop based on even or odd index.
  12. ng-template: <ng-template></ng-template> - Defines a template that can be used to render content conditionally or repeatedly.
  13. ng-container: <ng-container></ng-container> - Provides a structural element that is used to group and manipulate multiple elements.
  14. ng-content: <ng-content></ng-content> - Represents projected content within a component's view.
  15. @ViewChild: @ViewChild('elementRef') elementRef: ElementRef - Obtains a reference to an element or directive in the component's view.
  16. @ContentChild: @ContentChild('elementRef') elementRef: ElementRef - Obtains a reference to an element or directive projected into the component.
  17. ngOnInit: ngOnInit() - Lifecycle hook that is called after the component has been initialized and its inputs have been set.
  18. ngOnDestroy: ngOnDestroy() - Lifecycle hook that is called when the component is about to be destroyed.
  19. ngAfterViewInit: ngAfterViewInit() - Lifecycle hook that is called after the component's view has been initialized and rendered.
  20. ngDoCheck: ngDoCheck() - Lifecycle hook that is called during every change detection run, allowing custom change detection logic.
